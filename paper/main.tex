\begin{abstract}
Adaptive computation promises to match transformer inference cost to input complexity, but existing methods---early exit classifiers, mixture-of-depths, layer pruning---either require auxiliary networks, from-scratch training with compute-aware losses, or irreversible post-hoc surgery that degrades quality.
We observe that \emph{Attention Residuals} (\attnres{})~\citep{fu2025attention}---a drop-in replacement for the standard residual that attends over previous block outputs with a learned pseudo-query---already encode the routing signal adaptive computation needs, and their two-phase execution makes this signal available \emph{before} each block runs, at near-zero extra cost.
We develop this observation into a practical recipe in three parts.
\textbf{(1)~\reskip{}}: we read the phase-1 \attnres{} weights as a pre-execution skip signal, with position-calibrated thresholds. On a 340M from-scratch \attnres{} transformer pretrained on FineWeb-Edu, \reskip{} achieves $1.19\times$ wall-clock speedup with \emph{zero} degradation across LAMBADA, HellaSwag, ARC-Easy, and ARC-Challenge. We also document an \emph{importance--ablation disconnect}: the most-referenced block is not the most irreplaceable one---combining both signals strictly dominates either alone.
\textbf{(2)~\retrofit{}} (the main contribution): we show how to convert \emph{any} pretrained transformer into an \attnres{}-capable model through a single light fine-tune, without destroying its existing quality. We compare three retrofit routes---interpolation, auxiliary observer via distillation, and informed-initialization temperature annealing---and characterise when each works.
\textbf{(3)~VLA application}: retrofitting a vision-language-action backbone yields modality-specific routing that translates into modality-aware skipping for robotic inference.
Together, these give a path from an off-the-shelf transformer to an adaptive-depth model in a single fine-tune pass, with no architectural changes at inference.
\end{abstract}

\section{Introduction}
\label{sec:intro}
%----------------------------------------------------------------------

Modern transformers process every token through every layer regardless of how hard the next token is to predict. Several lines of work try to fix this mismatch: early-exit methods~\citep{schuster2022confident, elbayad2020depth} add auxiliary classifiers per layer, Mixture-of-Depths~\citep{raposo2024mixture} learns per-token routing with an auxiliary capacity loss, layer-pruning approaches~\citep{gromov2024unreasonable, men2024shortgpt} remove layers post-hoc, and Universal Transformers~\citep{dehghani2018universal} share weights across depth with explicit halting~\citep{graves2016adaptive}. All of these share a limitation: the routing signal---whether to spend more compute---is \emph{external} to the core transformer computation and must be added either as training-time supervision, a separate head, or a post-hoc heuristic.

\paragraph{Insight.}
Attention Residuals (\attnres{})~\citep{fu2025attention} replace the fixed scalar-$1$ residual with a learned softmax attention over all previously completed block outputs. The attention weights $\alpha_{n \to l}$ are input-dependent, competitively normalised, and---crucially for this paper---are computed \emph{before} block $l$ runs, as part of forming the block's input. Two observations follow. First, the weights already constitute a natural routing signal: a block that downstream blocks never attend to is, by construction, a block that can be skipped. Second, this signal is available as a pre-execution check, at negligible cost, so we can make the skip decision \emph{at inference time}, per input, without additional parameters or training.

\paragraph{But \attnres{} must be trained in.}
\attnres{} is an architectural modification: the pseudo-queries $\{\mathbf{w}_l\}$ and the online-softmax merge (Algorithm~\ref{alg:two_phase}) have no counterpart in a standard residual network. Existing works on \attnres{}---including its introduction~\citep{fu2025attention} and our own from-scratch validation---assume the model was pretrained with \attnres{} from step zero. This is expensive: reproducing a 7B-scale \attnres{} model costs tens of thousands of GPU-hours and forfeits all the investment in existing checkpoints. If adaptive computation via \attnres{} is to have practical value, we need a way to \emph{retrofit} pretrained transformers into \attnres{}-capable ones, cheaply, without destroying their quality.

\paragraph{Contributions.}
This paper delivers that retrofit recipe, along with its downstream application to skipping and to vision--language--action models.

\begin{enumerate}
\item \textbf{\reskip{} (Section~\ref{sec:reskip}).} We formalise the two-phase execution of \attnres{} as an implicit pre-execution routing channel and turn it into a dynamic, per-input skip rule with position-calibrated thresholds. On 340M from-scratch \attnres{} we achieve \textbf{$1.19\times$ wall-clock speedup at zero benchmark degradation}. We further identify an \emph{importance--ablation disconnect}: the \attnres{} importance score (``how often is block $n$ referenced?'') does \emph{not} align monotonically with static removal impact (``is block $n$ irreplaceable?''). Combining the two signals to pick skip candidates strictly dominates using either alone.

\item \textbf{\retrofit{} (Section~\ref{sec:retrofit}, main contribution).} We give three concrete retrofit routes and a unified training protocol:
  \begin{itemize}
  \item[\routeA{}] \emph{Interpolation gate}. Introduce a learned scalar $\beta_l$ that convexly mixes the standard residual output with the \attnres{} output. Initialised at $\beta_l{=}0$, the retrofit starts exactly at the pretrained model.
  \item[\routeB{}] \emph{Auxiliary observer + distillation}. Keep the forward pass standard. Train pseudo-queries so that the implied \attnres{} output matches the true forward; $\alpha$ becomes a learned importance map over the frozen network.
  \item[\routeC{}] \emph{Informed initialisation with temperature annealing}. Seed $\mathbf{w}_l$ from calibration statistics of the pretrained model's keys and anneal the softmax temperature so that \attnres{} starts near a residual-like regime and relaxes towards full expressivity.
  \end{itemize}
  The three routes trade off implementation complexity, parameter count, and inference cleanliness; we characterise each empirically.

\item \textbf{VLA application (Section~\ref{sec:vla}).} Applying the retrofit recipe to a 2B-scale vision--language--action backbone surfaces interpretable modality-specific routing (vision tokens prefer early blocks, action tokens use late blocks), and the resulting \emph{modality-aware skip} saves compute relative to uniform skip at matched task success rate.
\end{enumerate}

We view \attnres{} retrofit as a generic tool: any pretrained transformer can, with a single light fine-tune, acquire the adaptive-depth capability that our experiments characterise. Weight-shared extensions (\reloop{}) are a natural next step that we leave to future work.

\section{Background: Attention Residuals}
\label{sec:background}
%----------------------------------------------------------------------

\subsection{Standard residual connections}
In a standard transformer with $L$ layers,
\begin{equation}
  \mathbf{x}_l = \mathbf{x}_{l-1} + f_l(\mathbf{x}_{l-1}), \qquad
  \mathbf{x}_L = \mathbf{x}_0 + \sum_{l=1}^{L} f_l(\mathbf{x}_{l-1}).
\end{equation}
The combination weights are fixed at $1$ for every layer and every input---no routing information is emitted.

\subsection{Attention Residuals}
\attnres{}~\citep{fu2025attention} replaces the fixed combination with a learned attention over depth. Each block $l$ maintains a pseudo-query $\mathbf{w}_l \in \R^d$; keys are projections $\mathbf{k}_i = W_K \mathbf{h}_i$ of earlier block outputs $\mathbf{h}_i$:
\begin{equation}
  \alpha_{i \to l} = \frac{\exp(\mathbf{w}_l^\top \mathbf{k}_i / \sqrt{d})}{\sum_{j=0}^{l-1} \exp(\mathbf{w}_l^\top \mathbf{k}_j / \sqrt{d})},
  \qquad
  \mathbf{x}_l = \sum_{i=0}^{l-1} \alpha_{i \to l}\,\mathbf{h}_i.
  \label{eq:attnres}
\end{equation}
For efficiency, block-\attnres{} groups layers into $N$ blocks and applies the softmax at the block level, which is the granularity we use throughout.

\paragraph{Two-phase execution.}
A subtle but crucial property: computing the input to block $l$ requires only $\{\mathbf{h}_i\}_{i<l}$ and $\mathbf{w}_l$, both of which are available \emph{before} block $l$ runs. We refer to this as \emph{phase~1} (pre-execution attention over completed blocks). \emph{Phase~2} is the block's own transformation $f_l$; its output is folded into the running online-softmax state to produce the input of the next block. Algorithm~\ref{alg:two_phase} presents the unified two-phase forward, with \reskip{}'s dynamic skip check introduced at line~6.

\begin{algorithm}[t]
\caption{Two-phase \attnres{} forward with dynamic \reskip{}}
\label{alg:two_phase}
\begin{algorithmic}[1]
\STATE \textbf{Input:} Embeddings $\mathbf{h}_0$; block functions $f_1, \ldots, f_N$; pseudo-queries $\{\mathbf{w}_n\}$; eligible positions $\mathcal{P}$; thresholds $\{\tau_n\}_{n\in\mathcal{P}}$; max skips $M_{\max}$.
\STATE Initialise online-softmax state $(\mathbf{s}, m, e) \gets \textsc{Init}(\mathbf{h}_0)$; skip counter $k \gets 0$.
\FOR{$n = 1$ to $N$}
  \STATE \textbf{Phase 1 (probe).} Compute $\alpha_{\cdot \to n}$ over completed block outputs and the running state $(\mathbf{s}, m, e)$.
  \STATE Extract $w_{\text{recent}}(n) \gets \E_{\text{token}}[\alpha_{n-1 \to n}]$.
  \IF{$n \in \mathcal{P}$ \textbf{and} $k < M_{\max}$ \textbf{and} $w_{\text{recent}}(n) > \tau_n$}
    \STATE \textbf{Skip} block $n$; keep state unchanged, set $\mathbf{h}_n \gets \mathbf{h}_{n-1}$, $k \gets k+1$.
  \ELSE
    \STATE \textbf{Execute} block $n$ on the phase-1 merged input; update $(\mathbf{s}, m, e)$ with the new output $\mathbf{h}_n$.
  \ENDIF
\ENDFOR
\STATE \textbf{Output:} final hidden state $\mathbf{s}/e$.
\end{algorithmic}
\end{algorithm}

\section{\reskip{}: AttnRes-Guided Layer Skipping}
\label{sec:reskip}
%----------------------------------------------------------------------

\reskip{} turns the phase-1 signal of \attnres{} into a per-input skip rule. We first describe the method (Section~\ref{sec:reskip_method}), then expose a finding about block-level importance that informs eligible-position selection (Section~\ref{sec:reskip_disconnect}), and finally validate the approach on a 340M from-scratch \attnres{} model (Section~\ref{sec:reskip_results}).

\subsection{Method}
\label{sec:reskip_method}

\paragraph{Two offline scores per block.}
We characterise each block~$n$ with two complementary quantities computed on a small calibration set:
\begin{align}
I(n) &= \max_{l > n}\,\E\!\left[\alpha_{n \to l}\right]
  && \text{(\emph{\attnres{} importance}: how often block $n$ is referenced)} \label{eq:importance} \\
A(n) &= \frac{\text{PPL}(\text{model with block $n$ statically removed})}{\text{PPL}(\text{full-depth})}
  && \text{(\emph{ablation impact}: is block $n$ irreplaceable?)} \label{eq:ablation}
\end{align}
$I(n)$ requires only forward passes; $A(n)$ requires $N-2$ ablated forward passes. Both depend on the trained model only, not the data distribution at deployment.

\paragraph{Dynamic skip from phase-1 statistics.}
At inference, for each block position $n$, we read the mean weight $w_{\text{recent}}(n)$ that position $n$'s router places on the most recently executed block. The default skip rule \texttt{recent\_weight\_gt} is:
\begin{equation}
  \text{skip block $n$} \iff
  w_{\text{recent}}(n) > \tau_n
  \;\;\text{and}\;\;
  n \in \mathcal{P}
  \;\;\text{and}\;\;
  \textstyle\sum_{m<n} \mathbb{1}[\text{skipped}_m] < M_{\max}.
  \label{eq:skip_rule}
\end{equation}
The intuition is causal: if the next router already wants to copy the most recent output, block $n$ is doing local refinement that the model does not rely on.

\paragraph{Position-specific calibrated thresholds.}
A single global $\tau$ is sub-optimal; the distribution of $w_{\text{recent}}(n)$ differs sharply across positions. We calibrate per position on $32$ held-out long-context sequences, setting $\tau_n$ to the $q$-th empirical quantile of $w_{\text{recent}}(n)$ ($q{=}0.85$ by default). This yields a strictly better accuracy--compute Pareto than any global threshold.

\paragraph{Eligible-position selection uses both $I$ and $A$.}
Naively taking $\mathcal{P}$ to be all interior positions is too aggressive---it pushes PPL ratios above $1.3$ even at mild quantiles (Figure~\ref{fig:reskip_pareto}). Prior analyses tend to pick $\mathcal{P}$ by low $I$ alone; we show (Section~\ref{sec:reskip_disconnect}) that one must combine both scores:
\begin{itemize}
  \item \textbf{Low $I(n)$} $\Rightarrow$ skip triggers \emph{frequently} (the next router genuinely rarely uses block $n$);
  \item \textbf{Low $A(n)$} $\Rightarrow$ skip is \emph{safe} (even a mis-trigger preserves quality).
\end{itemize}

\paragraph{Cost of the skip decision.}
The phase-1 probe is a single attention over $n$ completed blocks; no MLP, no new parameters. The decision rule is a scalar comparison. Calibration produces $\tau_n$ in a single 32-batch pass and is embedded in the checkpoint's \texttt{config.json}; downstream users simply load and run. No architectural changes, no additional parameters, no training modifications.

The striking observation is that $I(n)$ and $A(n)$ \emph{disagree on a case-by-case basis}. Table~\ref{tab:reskip_benchmark} shows that both the single-position and combined-position configurations preserve every benchmark metric exactly. Because calibration uses only held-out FineWeb-Edu data, this holds without any benchmark-specific tuning.

\section{\retrofit{}: Retrofit Method}
\label{sec:retrofit}
%----------------------------------------------------------------------

\subsection{Problem statement}
Let $\theta^\star$ be the parameters of a transformer pretrained in the standard-residual regime, reaching some quality level $Q_0$ on a benchmark suite. We want to produce parameters $\theta'$ of an \attnres{} model such that:
\begin{description}
  \item[(P1)~\emph{Quality preservation:}] the retrofit model matches $\theta^\star$'s benchmark quality to within a small tolerance (we target $\leq 2\%$ mean drop).
  \item[(P2)~\emph{Routing informativeness:}] the learned pseudo-queries $\{\mathbf{w}_l\}$ produce an $\alpha$ distribution that is informative enough to drive a non-trivial \reskip{} Pareto (e.g., $\geq 20\%$ compute saved at $\leq 5\%$ PPL drop).
  \item[(P3)~\emph{Cheap:}] the fine-tune cost is small compared to pretraining, and uses off-the-shelf data.
\end{description}

\paragraph{Why this is not trivial.}
A single pseudo-query $\mathbf{w}_l$ cannot, for arbitrary inputs, make $\alpha_{l-1 \to l} = 1$ exactly, so \attnres{} cannot precisely reproduce the standard residual at initialisation. Naively swapping the residual for \attnres{} with random $\mathbf{w}_l$ immediately shifts every block's input and destroys the pretrained weights' calibration. We need retrofit strategies that either (a)~start at the pretrained model's exact forward and move off it gently, or (b)~leave the forward unchanged and learn $\mathbf{w}_l$ as an analytical observer.

\subsection{\routeA{}: Interpolation gate}
\label{sec:route_a}

Replace each block's residual with a convex combination of the standard and \attnres{} outputs controlled by a learned scalar $\beta_l \in [0,1]$:
\begin{equation}
  \mathbf{x}_l = (1 - \sigma(\beta_l)) \cdot \underbrace{\bigl[\mathbf{x}_{l-1} + f_l(\mathbf{x}_{l-1})\bigr]}_{\text{standard residual}}
  + \sigma(\beta_l) \cdot \underbrace{\sum_i \alpha_{i \to l}\,\mathbf{h}_i}_{\text{\attnres{}}},
  \label{eq:route_a}
\end{equation}
where $\sigma$ is the logistic function and $\beta_l$ is initialised at a large negative value (e.g., $-6$) so that $\sigma(\beta_l) \approx 0$ at the start of training: the model reproduces the standard residual exactly at step zero. Pseudo-queries $\mathbf{w}_l$ are initialised with small normal noise, yielding near-uniform $\alpha$ that does not yet affect the forward. Over fine-tuning, $\beta_l$ and $\mathbf{w}_l$ are learned jointly with a language-modelling loss on a small general-purpose corpus.

\textbf{Pros.} Guaranteed identity at initialisation; no catastrophic loss spike; stable training.
\textbf{Cons.} Final forward depends on \emph{both} paths, so skipping is conceptually cleaner but not purer. At inference, if $\sigma(\beta_l)$ stabilises at a small value, \reskip{}'s skip still saves the \attnres{} branch's cost; the standard residual path itself cannot be skipped without further training.

\subsection{\routeB{}: Auxiliary observer + distillation}
\label{sec:route_b}

Leave the pretrained forward untouched. Attach \attnres{} as an \emph{auxiliary head} that observes the completed block outputs $\{\mathbf{h}_i\}$ and is trained to reconstruct the true downstream input via:
\begin{equation}
  \mathcal{L}_{\text{obs}} = \sum_{l=1}^{L} \left\| \bigl(\sum_i \alpha_{i \to l; \mathbf{w}_l} \mathbf{h}_i\bigr) - \mathbf{x}_l \right\|_2^2.
  \label{eq:route_b}
\end{equation}
Only $\{\mathbf{w}_l\}$ (and optionally a key projection $W_K$) are trained; the rest of the network is frozen. At inference, we replace the standard residual with the learned \attnres{} output---the distillation loss encourages the two to match on the fine-tune distribution.

\textbf{Pros.} Original model is untouched during fine-tune; parameter count added is tiny ($\sim$$L d$); fastest wall-clock retrofit; $\alpha$ has a clean interpretation as ``how much did block $i$ actually contribute?''
\textbf{Cons.} Forward match is only approximate; \reskip{} quality depends on how tightly the observer learned. The skip decision is \emph{open-loop}: skipping does not feed back into training.

\subsection{\routeC{}: Informed initialisation + temperature annealing}
\label{sec:route_c}

Replace the forward with pure \attnres{}, but initialise intelligently. On a small calibration set, compute the average key $\bar{\mathbf{k}}_{l-1} = \E[\mathbf{k}_{l-1}]$ at each block and set $\mathbf{w}_l \gets c \cdot \bar{\mathbf{k}}_{l-1}$ for a large constant $c \gg 1$. This makes $\alpha_{l-1 \to l}$ close to $1$ in expectation---i.e., \attnres{} starts by approximately mimicking the standard residual. To avoid sharp attention at initialisation destabilising gradients, apply a temperature schedule:
\begin{equation}
  \alpha_{i \to l} = \softmax\!\left(\frac{\mathbf{w}_l^\top \mathbf{k}_i}{\sqrt{d}\,T(t)}\right), \qquad
  T(t) = T_{\text{start}} + (1 - T_{\text{start}}) \cdot \min(t / T_{\text{warmup}}, 1),
  \label{eq:route_c}
\end{equation}
annealing from $T_{\text{start}} \ll 1$ (sharp---approximately residual) to $T{=}1$ (full expressivity) over $T_{\text{warmup}}$ steps.

\textbf{Pros.} Final model is pure \attnres{}; conceptually cleanest; uses no auxiliary pathway at inference.
\textbf{Cons.} Informed init depends on calibration-set statistics and may transfer imperfectly to OOD inputs; temperature schedule is a sensitive hyperparameter; the highest-risk route.

\subsection{A unified retrofit recipe}
\label{sec:retrofit_recipe}

All three routes share the same training protocol, changing only the forward formula and initialisation:
\begin{enumerate}
  \item Start from pretrained checkpoint $\theta^\star$.
  \item Insert \attnres{} machinery per the chosen route (\routeA{}: gated mix; \routeB{}: observer head; \routeC{}: pure replacement with informed init).
  \item Fine-tune on a general-purpose corpus (we use the same FineWeb-Edu pipeline as Section~\ref{sec:reskip}) for $B$ tokens with a low learning rate ($\text{lr} \leq 10^{-5}$) and a short warmup; the LM loss is the standard next-token cross-entropy.
  \item Optionally add a light sparsity regulariser on $\alpha$ to encourage an informative Pareto (\texttt{SparseReg}: $\lambda \E[-\sum_i \alpha_{i \to l} \log \alpha_{i \to l}]$ with small $\lambda$).
  \item Calibrate $\{\tau_n\}$ as in Section~\ref{sec:reskip} and deploy \reskip{}.
\end{enumerate}
The fine-tune budget $B$ is the key hyperparameter we sweep; we expect stronger retrofit quality at larger $B$ but diminishing returns beyond a certain scale-dependent point.

\paragraph{Theoretical remarks.}
\routeA{} strictly dominates \routeB{} at initialisation (identity vs.\ approximate) and is strictly more flexible than \routeC{} (convex gate vs.\ fixed schedule). \routeC{} is strictly cleaner than the other two at \emph{convergence} (pure \attnres{}). The empirical question is whether the flexibility of \routeA{} actually pays off, or whether \routeC{}'s cleanness---if it trains at all---produces better routing for \reskip{}.

\subsection{Phase 2: \retrofit{} on pretrained LMs}
\label{sec:exp_retrofit}

This is the main experimental section. We will retrofit \attnres{} onto a standard open pretrained transformer and compare the three routes from Section~\ref{sec:retrofit} head-to-head.

\paragraph{Target.}
We target Llama-2-7B~\citep{touvron2023llama2} as the primary retrofit target: widely used, open weights, and large enough to make retrofit a realistic use case. A secondary 1--2B-parameter target (chosen to match the VLA-backbone scale in Section~\ref{sec:vla}) will be retrofitted with the best route from the primary comparison.

\paragraph{Retrofit grid.}
Each of \routeA{}/\routeB{}/\routeC{} will be run at fine-tune budgets $B \in \{500\text{M}, 2\text{B}, 5\text{B}, 10\text{B}\}$ tokens of FineWeb-Edu (matching pretraining distribution as closely as possible). For each completed retrofit we will measure:
\begin{enumerate}
  \item \textbf{Quality preservation} via full \texttt{lm-eval-harness} (LAMBADA, HellaSwag, ARC, MMLU, OpenBookQA) relative to the unmodified Llama-2-7B baseline.
  \item \textbf{\reskip{} Pareto} via the 90-point sweep defined in Section~\ref{sec:reskip} (positions $\mathcal{P}$, max skips $M_{\max}$, quantile $q$).
  \item \textbf{Routing informativeness} via the entropy of $\alpha$ distributions across the fine-tune trajectory; we want to see structured, non-uniform routing emerge.
  \item \textbf{Wall-clock speedup} at representative sequence lengths (2k, 8k) on a single H100.
\end{enumerate}

\paragraph{Baselines.}
We will compare against (a)~unmodified Llama-2-7B (full-depth, no skip), (b)~static pruning~\citep{gromov2024unreasonable}, (c)~LayerSkip~\citep{elhoushi2024layerskip} continued-pretraining, and (d)~Mixture-of-Depths~\citep{raposo2024mixture} where applicable.

\paragraph{Expected outcome.}
Our prior is that \routeA{} is the most likely to satisfy (P1) thanks to the identity-at-init property, that \routeB{} is the most parameter-efficient but will produce weaker routing (open-loop), and that \routeC{} will be the highest-variance in terms of whether training succeeds but will---when it does---produce the cleanest downstream \reskip{} Pareto. Final recommendation will be made on the empirical evidence.

\section{VLA Application}
\label{sec:vla}
%----------------------------------------------------------------------

\subsection{Motivation}
In Vision-Language-Action (VLA) models, tokens of different modalities serve fundamentally different roles:
\begin{itemize}
  \item \textbf{Vision tokens} encode perceptual features; prior work shows these stabilise in early layers~\citep{raghu2021vision}.
  \item \textbf{Language tokens} carry task specifications and reasoning.
  \item \textbf{Action tokens} require compositional motor planning and typically benefit from deeper processing.
\end{itemize}
There is no principled reason for all three modalities to use the same effective depth. Yet in practice all existing open VLA architectures~\citep{brohan2023rt, team2024octo, kim2024openvla} process every token through every layer. \attnres{}'s per-position routing offers a mechanism to tailor effective depth per modality without any token-level gating machinery.

\subsection{Planned experiments}
\label{sec:vla_plan}

\paragraph{Target VLA backbone.}
A 2B-scale open VLM (e.g., Qwen2-VL-2B) with frozen vision encoder, retrofitted via the best-performing route identified in Section~\ref{sec:exp_retrofit}. A flow-matching action head~\citep{black2024pi0} is attached for continuous control; the full model is fine-tuned on LIBERO-90~\citep{liu2023libero} and a subset of OpenX-Embodiment~\citep{embodiment2024openx}.

\paragraph{Analysis: modality-specific routing.}
For each held-out LIBERO trajectory we extract the per-token $\alpha$ distribution and average by modality. We expect (hypothesis H1) vision-token $\alpha$ to concentrate on early blocks, (H2) action-token $\alpha$ to spread to late blocks, and (H3) language-token $\alpha$ to sit between. Visualisations will mirror Figure~\ref{fig:reskip_routing}(a) per-modality.

\paragraph{Modality-aware skip.}
At inference, we allow different skip policies per modality:
\begin{equation}
  \text{skip block $n$ for modality $m$} \iff
  w_{\text{recent}}^{(m)}(n) > \tau_n^{(m)}
  \;\;\text{and}\;\; n \in \mathcal{P}^{(m)},
\end{equation}
with per-modality $(\mathcal{P}^{(m)}, \tau_n^{(m)}, M_{\max}^{(m)})$. A block is skipped for a token only if the rule fires under that token's modality, so aggressive settings on vision tokens do not starve action tokens. We compare against a uniform skip baseline at matched effective FLOPs.

\paragraph{Target metrics.}
LIBERO-90 success rate, action L1 error, inference FLOPs, wall-clock end-to-end latency. The main claim we aim to support: \emph{at matched success rate, modality-aware skip saves $\geq 15\%$ more FLOPs than uniform skip.}

\section{Related Work}
\label{sec:related}
%----------------------------------------------------------------------

\paragraph{Adaptive computation.}
ACT~\citep{graves2016adaptive} introduced learned halting for recurrent networks. Universal Transformers~\citep{dehghani2018universal} combine ACT with weight-shared transformer blocks but struggle to differentiate the role of each application of the same block. CALM~\citep{schuster2022confident} adds auxiliary classifiers per layer for early exit. Mixture-of-Depths~\citep{raposo2024mixture} routes individual tokens to different layer subsets with capacity losses. Our work differs by using the \attnres{} mechanism itself as the routing signal, without any auxiliary component---and, in the retrofit case, by installing that signal into an existing pretrained transformer.

\paragraph{Layer pruning and skipping.}
\citet{gromov2024unreasonable} showed that many transformer layers can be removed post-training with modest quality loss. ShortGPT~\citep{men2024shortgpt} introduces block-influence metrics. LayerSkip~\citep{elhoushi2024layerskip} combines early exit with self-speculative decoding. These approaches either do static removal (no input dependence) or require invasive training changes; our retrofit preserves the full network and adds only a light fine-tune.

\paragraph{Residual-connection innovations.}
DenseNet~\citep{huang2017densely} connects each layer to all previous via concatenation. Highway networks~\citep{srivastava2015highway} use gated residuals. \attnres{}~\citep{fu2025attention} generalises these with an attention-based weighting, which we adopt. We are the first to exploit \attnres{} weights as computation-routing signals and the first to show that the mechanism can be retrofitted into pretrained models.

\paragraph{Vision-Language-Action models.}
RT-2~\citep{brohan2023rt} and Octo~\citep{team2024octo} established the VLA paradigm. OpenVLA~\citep{kim2024openvla} demonstrated generalist manipulation. Pi0~\citep{black2024pi0} integrated flow-matching action heads. Efficient-VLA work so far has focused on action chunking~\citep{zhao2023learning} and post-hoc quantisation; we contribute the first modality-aware adaptive-depth mechanism for VLAs.

\section{Discussion}
\label{sec:discussion}
%----------------------------------------------------------------------

\paragraph{Summary.}
We show that \attnres{}'s two-phase execution already contains the routing signal adaptive computation needs; that this signal can be calibrated into a dynamic \reskip{} that preserves quality while saving $1.19\times$ wall-clock at 340M; and that \attnres{} can be \emph{retrofitted} into pretrained transformers through a light fine-tune, via one of three routes whose trade-offs we characterise.

\paragraph{Limitations.}
Our retrofit experiments (Section~\ref{sec:exp_retrofit}) are scheduled but not yet complete at the scale of Llama-2-7B; final recommendations between \routeA{}/\routeB{}/\routeC{} await those runs. The VLA evaluation (Section~\ref{sec:vla}) is similarly forward-looking. \reskip{}'s speedup is most visible at long sequence lengths; for short benchmark inputs, probe overhead is a larger fraction of per-block cost and the speedup shrinks correspondingly. The method inherits \attnres{}'s block-level granularity; per-token routing remains future work, complementary to (and potentially stackable with) ours.

\paragraph{Future work: weight-shared AttnRes (\reloop{}).}
A natural extension is to share block weights across depth and use position-indexed \attnres{} pseudo-queries to differentiate each application of the shared block. Unlike Universal Transformers, the pseudo-queries provide an explicit mechanism for ``first pass'' vs.\ ``third pass'' differentiation. Early validation on a 74M from-scratch model showed the training dynamics differ substantially from standard \attnres{}, and achieving competitive quality at Universal-Transformer parameter counts remains an open problem. Combined with our retrofit framework, \reloop{} offers a path to parameter-efficient adaptive-depth pretrained models; we leave this investigation to future work.

\paragraph{Broader impact.}
Retrofit reduces the compute cost of deploying adaptive-depth models: rather than re-running pretraining for every model one wants adaptive inference on, a single light fine-tune produces a skip-capable variant. This lowers the barrier to deploying efficient inference on existing models across the research and deployment ecosystem.

\section{Implementation details}
\label{app:implementation}
%----------------------------------------------------------------------

\subsection{Online softmax merge with skip}
When a block is skipped, the online softmax merge simply does not process its output. The running state $(\mathbf{s}, m, e)$ is unchanged. The output is mathematically equivalent to an \attnres{} model trained without that block contributing---modulo the calibration of downstream $\mathbf{w}_l$.

\subsection{Pseudo-query initialisation (Section~\ref{sec:reskip} from-scratch)}
For from-scratch training we use $\mathbf{w}_l \sim \mathcal{N}(0, 0.02)$, yielding near-uniform $\alpha$ at initialisation that specialises over the first few thousand steps.

\subsection{Retrofit hyperparameters}
Across all three routes we use FineWeb-Edu~100BT, AdamW, $\text{lr}{=}5\!\times\!10^{-6}$, linear warmup over 200 steps, context length 2k. \routeA{}'s gate $\beta_l$ initial value is $-6$ (so $\sigma(\beta_l) < 0.005$). \routeB{}'s observer loss weight is $\lambda_{\text{obs}}{=}1$; the LM loss is retained during fine-tune to prevent degenerate observer solutions. \routeC{}'s temperature schedule uses $T_{\text{start}}{=}0.05$, $T_{\text{warmup}}{=}500$ steps. Full settings will be released in a companion code repository.

\subsection{Block granularity}
We recommend $N = 6$--$8$ blocks for models with 12--32 layers, matching the settings in Section~\ref{sec:reskip}. For retrofit targets with different layer counts, block boundaries should match the pretrained model's natural stage boundaries (e.g., Llama-2-7B's 32 layers split into 8 blocks of 4).

\subsection{Calibration set}
Calibration uses 32 held-out long-context sequences ($\geq$8k tokens each) from the fine-tune distribution. We verified on 340M that changing the calibration set to a different 32-sequence draw shifts $\tau_n$ by less than $0.02$ in absolute terms, and does not change the recommended $\mathcal{P}$.
